## 3. 同类问题的典型症状分类

### 3.1 前端 API 404（契约错配）

表现：

- 浏览器 Network 中出现 `/api/...` 返回 `404 Not Found`
- 前端页面出现连锁 JS 报错（表格对象为空、渲染异常）

典型根因：

- 默认前端路径与后端 `router/api-router.go` 不一致
- 前端仍调用历史接口名，后端已改为新版路由

### 3.2 旧版 `/console/*` 路由残留

表现：

- 点击菜单后跳转到 `/console/...`
- 新版前端无对应旧路由，最终显示 404

典型根因：

- 迁移过程中菜单/跳转地址未切到默认前端路由体系

### 3.3 OIDC 登录链路 `invalid_client`

表现：

- 第三方项目走 `/oauth/authorize` 后，`/oauth/token` 返回 `401 invalid_client`

典型根因：

- provider 端与 consumer 端 `client_secret` 不一致
- provider 重建后密钥漂移（`CRYPTO_SECRET`/`SESSION_SECRET` 未固定）
- 缺少可视化 OIDC client 管理页，导致 secret 轮换与同步流程不闭环

### 3.4 签到“功能存在但配置面不完整”

表现：

- 用户侧签到入口存在，但管理端只有简化字段
- 三档签到（entry/basic/advanced）参数无法完整配置

典型根因：

- 默认前端设置页仍是旧字段模型，未对齐后端当前配置结构

## 4. 本次修复动作（可复用模板）

### 4.1 补齐默认前端 OIDC Provider 客户端管理页

新增模块路径：

- `web/default/src/features/system-settings/auth/oidc-hub-clients/`

覆盖能力：

- Client 列表：`GET /api/oidc/clients/`
- 新建 Client：`POST /api/oidc/clients/`
- 更新 Client：`PUT /api/oidc/clients/{client_id}`
- 轮换 Secret：`POST /api/oidc/clients/{client_id}/rotate_secret`
- 启用/禁用：`POST /api/oidc/clients/{client_id}/enable|disable`
- 删除：`DELETE /api/oidc/clients/{client_id}`
- Signing Key 列表：`GET /api/oidc/signing-keys/`
- 轮换 Signing Key：`POST /api/oidc/signing-keys/rotate`
- 激活 Signing Key：`POST /api/oidc/signing-keys/{kid}/activate`

挂载位置：

- `web/default/src/features/system-settings/auth/section-registry.tsx`

## 5. 下次遇到同类问题的标准处理顺序

按以下顺序执行，不要跳步：

1. **先分类错误类型**
   - 是 API 404、旧路由 404、还是 OIDC token 阶段 `invalid_client`

2. **先对齐前后端契约再改页面**
   - 前端 feature 请求路径
   - 后端 `router/api-router.go` 实际路由
   - 对应 controller 入参/返回结构

3. **优先修复“缺页/缺能力”而非只兜底提示**
   - 对于 OIDC，必须有 client 管理闭环（创建/轮换/同步/启停）

4. **OIDC 问题必须执行密钥一致性检查**
   - Provider `.env` 固定 `SESSION_SECRET` / `CRYPTO_SECRET`
   - Consumer `.env` 同步最新 `client_secret`
   - 用假 code 调 `/oauth/token`：
     - `invalid_grant` => secret 已对齐
     - `invalid_client` => secret 仍未对齐

5. **构建与后端验证必须完成**
   - 前端：`web/default` 构建通过
   - 后端：`go test ./...` 通过
   - 二进制：`go build -o new-api.exe .` 通过

6. **最后再做浏览器实测回归**
   - 优先回归二开高风险菜单与登录链路
