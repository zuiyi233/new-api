# 站点自定义与外链配置说明

> 更新时间：2026-04-15  
> 适用范围：当前仓库 `new-api` 代码与已公开官方资料  
> 目的：说明本项目当前支持哪些“超链接挂载 / 品牌展示自定义”能力，哪些需求可以只靠后台配置完成，哪些需求需要改源码。

---

## 1. 结论摘要

### 1.1 可以直接做的事情

当前项目已经支持通过后台配置，在以下位置添加可点击内容或自定义展示：

- 首页内容 `HomePageContent`
- 关于页 `About`
- 页脚 `Footer`
- 公告 `Notice`
- 系统公告 `console_setting.announcements`
- 文档链接 `general_setting.docs_link`
- 充值链接 `TopUpLink`
- 系统名称 `SystemName`
- Logo 图片地址 `Logo`

这些能力足以覆盖大多数场景，例如：

- 在首页或关于页放自己的网站链接
- 在页脚放官网、店铺、联系方式
- 在公告中放活动页或购买页链接
- 自定义站点标题、Logo、favicon、登录页品牌展示

### 1.2 不能直接做、通常需要改源码的事情

以下需求**不是当前后台直接支持的通用配置能力**：

- 新增任意顶部导航菜单项，例如“闲鱼店铺”“官网”“商务合作”
- 新增任意侧边栏菜单项
- 修改顶部 Logo 的点击行为（例如点击后跳到外部站）
- 将导航从“预置模块开关”改造成“任意菜单编辑器”

### 1.3 需要特别注意的边界

本仓库项目规则明确要求：

- 不得擅自删除、替换或移除 `new-api` / `QuantumNous` 相关受保护项目信息

因此本文仅说明：

- 如何**添加自己的展示内容**
- 如何**配置已有可自定义字段**
- 哪些地方**需要二次开发**

不包含“移除或替换受保护项目身份标识”的操作方案。

## 2. 后台可配置项总览

后台主要入口：

- **系统设置 → 其它设置**
- **系统设置 → 运营设置 / 通用设置**
- **系统设置 → 数据看板 / 系统公告设置**

### 2.1 其它设置中的核心个性化字段

文件依据：`web/src/components/settings/OtherSetting.jsx`

| 字段 | 作用 | 支持内容 | 备注 |
|---|---|---|---|
| `SystemName` | 系统名称 | 纯文本 | 影响站点标题、头部标题、登录页标题等 |
| `Logo` | Logo 图片地址 | 图片 URL | 影响头部 logo、登录页 logo、favicon（运行时） |
| `HomePageContent` | 首页内容 | Markdown / HTML / `https://` 链接 | 如果填 `https://` 开头链接，则以前端 iframe 方式加载 |
| `About` | 关于页内容 | Markdown / HTML / `https://` 链接 | 如果填 `https://` 开头链接，则以前端 iframe 方式加载 |
| `Footer` | 页脚内容 | HTML | 非空时直接作为自定义页脚 HTML 渲染 |
| `Notice` | 公告 | Markdown / HTML | 首页弹出公告内容 |

### 2.2 运营设置中的相关外链项

文件依据：`web/src/pages/Setting/Operation/SettingsGeneral.jsx`

| 字段 | 作用 | 典型用途 |
|---|---|---|
| `TopUpLink` | 充值链接 | 跳转到发卡站、支付页、购买页 |
| `general_setting.docs_link` | 文档地址 | 把“文档”入口指向你的帮助站或外部说明页 |

### 2.3 顶部导航模块配置

文件依据：`web/src/pages/Setting/Operation/SettingsHeaderNavModules.jsx`

当前顶部导航只支持预置模块开关，模块包括：

- `home`
- `console`
- `pricing`
- `docs`
- `about`

这说明：

- 可以**控制已有模块是否显示**
- 不能通过后台**新增任意自定义模块**

## 3. 各位置的行为说明

## 3.1 首页内容 `HomePageContent`

文件依据：

- `web/src/components/settings/OtherSetting.jsx`
- `web/src/pages/Home/index.jsx`
- `controller/misc.go`

相关行为：

1. 前端会请求接口：`GET /api/home_page_content`
2. 若返回内容**不是** `https://` 开头：
   - 使用 `marked.parse(...)` 解析为 Markdown/HTML
   - 再渲染到页面
3. 若返回内容是 `https://` 开头：
   - 直接作为 `<iframe src="...">` 的内容来源

### 推荐用途

- 品牌欢迎页
- 官网入口
- 活动页入口
- 带按钮的落地页

### 示例 1：使用 Markdown 放外链

```md
# 欢迎来到我们的服务

- 官网：[点击访问](https://example.com)
- 店铺：[点击前往](https://shop.example.com)
- 联系邮箱：hello@example.com
```

### 示例 2：使用 HTML 放按钮

```html
<div style="padding: 32px; text-align: center;">
  <h1>欢迎来到我们的服务</h1>
  <p>这里可以放介绍、指引和外链按钮。</p>
  <p>
    <a href="https://example.com" target="_blank" rel="noopener noreferrer">访问官网</a>
    &nbsp;|&nbsp;
    <a href="https://shop.example.com" target="_blank" rel="noopener noreferrer">前往店铺</a>
  </p>
</div>
```

### 示例 3：直接使用 iframe 页面

```text
https://example.com/landing
```

### 注意事项

- 代码当前只对 **`https://`** 做 iframe 识别
- 若目标站点禁止 iframe 嵌入，则页面无法正常展示
- 第三方平台页面通常更适合**超链接**而不是 iframe

## 3.3 页脚 `Footer`

文件依据：

- `web/src/components/settings/OtherSetting.jsx`
- `web/src/components/layout/Footer.jsx`
- `controller/misc.go`

行为：

- 读取 `footer_html`
- 若非空，则直接通过 `dangerouslySetInnerHTML` 渲染你的自定义页脚 HTML

### 推荐用途

- 官网链接
- 店铺链接
- 联系方式
- 备案信息
- 辅助说明

### 示例

```html
<div style="display:flex;gap:12px;flex-wrap:wrap;align-items:center;">
  <span>© 2026 我的品牌</span>
  <a href="https://example.com" target="_blank" rel="noopener noreferrer">官网</a>
  <a href="https://shop.example.com" target="_blank" rel="noopener noreferrer">店铺</a>
  <a href="mailto:hello@example.com">联系我们</a>
</div>
```

### 重要说明

即使自定义页脚已生效，当前前端实现中仍保留一个默认的品牌归属展示区。是否进一步调整，属于源码修改范畴，且需遵守项目规则与授权边界。

- 登录 / 注册 / 重置密码页标题

## 4.2 Logo `Logo`

文件依据：

- `web/src/components/settings/OtherSetting.jsx`
- `web/src/components/layout/PageLayout.jsx`
- `web/src/components/layout/headerbar/HeaderLogo.jsx`

应用运行后，前端会根据配置动态覆盖这些值；但如果追求“页面初次加载前也完全替换”，则需要源码级处理。

## 5. 最适合放外链的位置建议

如果你的目标只是给用户一个稳定、清晰的跳转入口，推荐优先级如下：

### 第一优先级：`Footer`

适合：

- 常驻官网链接
- 店铺链接
- 联系方式

优点：

- 全站可见
- 不影响主流程
- 适合长期展示

### 第二优先级：`About`

适合：

- 品牌介绍
- 详细说明
- 外链集合页

### 第三优先级：`HomePageContent`

适合：

- 自定义首页
- 活动页入口
- 更强的品牌化展示

### 第四优先级：`Notice` / `announcements`

适合：

- 临时活动
- 购买引导
- 限时跳转

## 6. 哪些需求需要改源码

以下需求若要实现，应视为前端/配置结构扩展任务：

### 6.1 新增任意顶部导航入口

可能涉及文件：

- `web/src/hooks/common/useNavigation.js`
- `web/src/components/layout/headerbar/Navigation.jsx`
- `web/src/pages/Setting/Operation/SettingsHeaderNavModules.jsx`

### 6.2 新增任意侧边栏菜单

可能涉及文件：

- `web/src/components/layout/SiderBar.jsx`
- 相关权限与模块配置文件

### 6.3 修改 Logo 点击跳转行为

当前头部 Logo 组件使用的是站内首页链接，若要跳外部站，通常需要修改：

- `web/src/components/layout/headerbar/HeaderLogo.jsx`

### 6.4 扩展“任意自定义链接配置”

如果希望后台直接支持：

- 菜单名称
- 链接地址
- 打开方式
- 是否展示
- 图标

则需要新增配置结构、后台设置表单与前端导航消费逻辑。

## 7. 风险与限制

## 7.1 iframe 限制

虽然首页与关于页支持把 `https://` 内容当 iframe 使用，但并不是所有网站都允许被嵌入。

常见失败原因：

- `X-Frame-Options`
- Content Security Policy (CSP)
- 第三方平台防嵌入策略

因此：

- 自己可控的网站：可优先尝试 iframe
- 第三方网站 / 店铺页：优先使用普通超链接

## 7.2 HTML 注入风险

当前这些可配置内容大量依赖：

- `marked.parse(...)`
- `dangerouslySetInnerHTML`

这意味着：

- 自定义能力很强
- 同时也意味着 HTML 风险由部署方自行控制

建议：

- 仅允许可信管理员修改这些字段
- 不要把这些配置开放给普通用户

## 7.3 受保护项目信息边界

根据本仓库项目规则，`new-api` / `QuantumNous` 相关项目身份信息受保护，不得擅自删除或替换。本文不提供超出该边界的实施说明。

## 8. 推荐实施方案

## 方案 A：仅用后台配置完成

适合需求：

- 放官网或店铺链接
- 加品牌介绍
- 自定义页脚
- 修改系统名与 Logo

推荐组合：

- `SystemName`
- `Logo`
- `Footer`
- `About`
- `HomePageContent`
- `Notice` / `announcements`

## 方案 B：需要新增导航能力

适合需求：

- 顶部新增一个“官网”或“店铺”菜单项
- 侧边新增“帮助中心”等入口

此时建议按“二次开发任务”处理，而不是把它当成纯配置操作。

## 10. 一句话总结

当前项目已经支持通过后台在**首页、关于页、页脚、公告**等位置放置可点击链接，并支持设置 **SystemName / Logo** 来完成大部分品牌展示自定义；但如果要**新增任意导航项、改 Logo 点击行为或扩展成完整菜单编辑器**，则需要改源码。
